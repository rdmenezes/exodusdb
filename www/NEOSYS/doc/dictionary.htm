Functions

Dictionary function are designed to quickly setup repetitive data types. Additional properties can be added and overridden after the dictionary item is setup.

function neosys_dict_index(di,filename,fieldname,many,allownew,warnnew)

Sets up a data field to popup/validate against indexed values

filename defaults to the current data entry file

fieldname defaults to the current dictionary fieldname

many default to true if the current dictionary fieldname is multivalued (has a groupno)

allownew defaults to true. if false then user cannot enter new values

warnnew defaults to true. if false then the user is warned if entering new values

function neosys_dict_year(dictitem,from,to,defaultyear)

Sets up a dictionary item to allow four digit years in a dropdown box.

dictitem is a dictionary item to be converted

from is an optional offset from the current year to start from

to is an optional offset from the current year to stop at

defaultyear is an optional offset from the current year otherwise there is no default

function neosys_dict_datetimeupdated(dictitem)

Sets up a dictionary item to show time and date and makes it read only and not copyable

function neosys_dict_datetime(dictitem)

Sets up a dictionary item to handle time and date in one field eg. "1/1/2004 12:59"

function neosys_dict_date(dictitem,params)

Sets up a dictionary item to handle date eg "1/1/2004"

The conversion is set as "[DATE,params]"

function neosys_dict_time(dictitem,mode,otherid,params)

Sets up a dictionary item to handle time eg "12:59"

mode "FROM" or "UPTO" indicates that this is one of a pair of time fields indicating a period of time

otherid the dictionary id of the "other" time field

conversion The conversion is set as "[TIME,params]" Will not overide an existing conversion on the dictitem.

function neosys_dict_period(dictitem)

Sets up a dictionary item to handle a PERIOD_OF_YEAR eg 1/2004

function neosys_dict_yesno(dictitem,defaultvalue,type)

Sets up a dictionary to handle a Yes/No field (internally "Y" or "N")

defaultvalue "Y" or "N" or defaults to "Y"

type "dropdown"=a drop down box, "radio"=vertical aligned, "hradio"=horizontal aligned

function neosys_dict_number(dictitem,params,params2)

Sets up a dictionary to handle numbers

params can be:

  • a digit for the number of decimals
  • "BASE" which indicates base format
  • "NDECS" in which case gndecs or getrecord('NDECS') used
  • append a Z for suppression of zeroes

params2 can be "POSITIVE" to restrict the entry to positive numbers

function neosys_dict_text(dictitem,size,rows)

Setups up a dictionary item to handle text

size will determine the approximate size of the data entry field. Default value is 30

rows will determine the depth (number of lines) of  the data entry field. Default value is 1

Lower case will be allowed
